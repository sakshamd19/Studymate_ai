# Microsoft Agents League Hackathon - Rule Compliance Audit

## Phase 0: System Compliance Map

### 1. Multi-agent system exists?
**Status:** PASS 
**Details:** The system utilizes an OrchestratorAgent that manages 11 distinct intelligence agents.

### 2. Agents have REAL roles (not wrappers)?
**Status:** FAIL (Partial)
**Details:** 
- Most agents have robust internal logic, but some are missing.
- **Missing Agents:** `EngagementAgent`, `ManagerInsightsAgent`.
- **Misaligned Agents:** `MockExamAgent` needs to be converted/renamed to `AssessmentAgent` to strictly evaluate readiness rather than just generating mock queries.

### 3. Microsoft IQ integration exists and is FUNCTIONAL?
**Status:** PASS / WEAK IMPLEMENTATION
**Details:** `FoundryIQClient` implements Azure AI Projects SDK integration. However, the system relies too heavily on generic synthetic fallbacks when live API is unavailable. Needs internal structural mimicking of Foundry IQ grounded retrieval when API is missing.

### 4. System uses ONLY synthetic data?
**Status:** PASS
**Details:** `input_normalizer.py` strictly prevents PII leakage and email/AADHAR insertion.

### 5. No PII leakage anywhere?
**Status:** PASS
**Details:** Handled by regex filters in the pipeline before orchestration.

### 6. Outputs are grounded (not hallucinated)?
**Status:** FAIL
**Details:** `agent_envelope` currently returns `reasoning_trace` and `confidence`, but strictly lacks an `evidence` array which is required by Phase 3 rules to prevent generic LLM hallucinations.

### 7. Agents are orchestrated (not sequential scripts)?
**Status:** PASS
**Details:** Agents are coordinated using `ThreadPoolExecutor` and dependency passing within `OrchestratorAgent`.

### 8. Reasoning is visible and explainable?
**Status:** FAIL (Partial)
**Details:** Reasoning traces are generated by the agents internally, but the UI must be updated to clearly expose "Agent Intelligence" in a dedicated tab as requested by the judges.

### 9. System flow matches required pipeline?
**Status:** FAIL
**Details:** The flow currently lacks the "Track Engagement" and "Recommend Next Step" nodes. It must strictly follow: Input -> Learning Path -> Schedule -> Engagement -> Assessment -> Insights.

**Details:** Missing the `đź§  Agent Intelligence` tab with consensus, counterfactuals, critic findings, and observability logs as explicitly demanded by the judges.
